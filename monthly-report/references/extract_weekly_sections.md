# 从「XX月工作」正文中提取「第 x 周周报」

## 标题形态

语雀中常见标题包括但不限于：

- `第1周周报`、`第2周周报`、`第3周周报`
- `第 1 周周报`（含空格）
- 中文数字：`第一周周报`（若出现，按同层级对待）

**正则参考**（去 HTML 后的纯文本上匹配）：

```text
第\s*[0-9一二三四五六七八九十]+\s*周\s*周报
```

## 推荐流程

1. 将 `.data.content` 转为可读文本（去掉 `<tag>`，保留换行；或将 `<h1>`~`<h4>` 替换为换行 + 原标题文本）。
2. 用上述标题模式在全文**搜索所有匹配位置**，按出现顺序排序。
3. **每个「第 x 周周报」块**：从本标题的下一行起，直到**下一个**同级周报标题（或文档中更高级的月份/周区间标题如 `03-23~03-27 第4周`）之前，作为该周素材。
4. **缺少周报表头**：若文档仅有「第 x 周」全文周报草稿而无「第 x 周周报」六字，可退化为：凡明确标注为当周周报的章节（如「第1周周报」下挂的「业务重点」整段），仍归入对应周；否则将文档后半部分按日期块人工划定周界时，在月报中说明依据。

## 与各周小节对应关系

单篇「第 x 周周报」内若出现与周报一致的小节名，可直接归类：

| 周报小节（语雀常见） | 月报归属 |
| --- | --- |
| 业务重点 / OBCloud 需求支持 | 汇总进 **一、OBCloud 业务支持** |
| 体验优化与创新 | 汇总进 **二、体验打磨与创新** |
| 其它 | 汇总进 **三、团队与其它** |

若某周周报**没有**二级标题，则将该周全文作为原始素材，由 Agent 按三块主题**自行归纳**，仍遵守月报格式（不按天罗列、不重复堆砌）。

## 与按日流水共存

「XX月工作」中常混有 **按日记录**（如 `03-18 周三`）与 **第 x 周周报**。**月报 skill 仅以「第 x 周周报」章节为输入**；若用户要求「把按日记录也算进月报」，需明确指令后再扩展范围。
